# ACE K주도주(가칭) — 이름 순위 산출 시뮬레이션 요구사항

## 배경

ACE K주도주(가칭)는 한국투자신탁운용이 2026년 상반기 상장을 준비 중인 국내주식 ETF다. 반도체·자동차·조선·방위산업·원자력 5대 섹터 × TOP2 = 10종목 정예 포트폴리오로, 반차조방원 슈퍼사이클을 한 바구니에 담는 상품이다. 지수 방법론은 ETF상품전략부 초안으로 확정되었으나, **최종 상품명은 미정**이다.

시드문서(k주도주(가칭)_시드문서.md)에는 13명의 투자자 에이전트(10개 유형)와 13개 이름 후보가 정의되어 있다. 본 시뮬레이션은 각 에이전트가 13개 후보 이름 각각에 어떻게 반응하는지를 MiroFish 멀티에이전트 엔진으로 병렬 추정하고, 유형별 가중 평균을 통해 **최종 순위(Final Ranking)와 선정 이유**를 도출하는 것을 목적으로 한다.

## 핵심 질문

1. 13개 이름 후보 중 리테일 투자자 전체 기준 1위 후보는 무엇이며, 그 이유는 무엇인가?
2. 투자자 유형별로 최선호 후보가 어떻게 달라지는가? 유형 간 합의 후보가 존재하는가?
3. 방법론 정합성(승인 심사)과 리테일 어필(바이럴·검색·매수 전환) 간 트레이드오프에서 최적점은 어디인가?
4. 바이럴 극단 후보(반차조방원)와 직관 극단 후보(반도체자동차플러스)는 방법론 친화 후보들과 비교해 얼마나 경쟁력이 있는가?
5. 정식 상품명 1개 + 서브 캠페인 해시태그 1개 + 숏폼용 밈 네임 1개의 3단 구조로 가면, 각 슬롯에 어떤 후보가 최적인가?
6. 이름 자체로 해결 불가능한 구조적 리스크(AI 부재, 반도체 50% 집중, KODEX 200 중복)는 어떻게 보완해야 하는가?

## 13개 이름 후보 (시드문서 참조)

후보는 시드문서 "이름 후보 리스트" 섹션에 정의되어 있다. 요약:
- AutoResearch 파생: K5대전략TOP10, K-5대전략TOP10, K-5대액티브TOP10, K-5대산업TOP10, K-5대주도TOP10, K-5전략TOP10, K-5대산업주도TOP10, K-5대코어TOP10, 코리아5대주도TOP10, K-5대K-TOP10
- 방법론 베이스라인: K5대산업 정예TOP10
- 사용자 추가(대조군): 반차조방원, 반도체자동차플러스

## 시뮬레이션 시나리오

### 시나리오 M (메인) — 이름 단독 평가

각 에이전트에게 13개 이름 후보를 순차 제시하고 첫인상·이해도·매수 의향을 측정한다. 컨텍스트 노이즈를 최소화한 순수 이름 반응 베이스라인. 각 후보에 대한 에이전트 반응을 ◎(3)·○(2)·△(1)·✕(0) 4단계로 기록하고, 10개 유형 가중치(연금 적립 35%, MTS 직접매매 25%, PB 자문 8%, 시니어 첫 투자 12%, 기타 20%)로 종합 점수를 산출한다.

### 시나리오 C (채널) — 채널별 노출 맥락 차이

같은 이름이라도 노출 채널에 따라 반응이 달라진다. 다음 4개 채널 맥락에서 각 이름의 호감도를 재측정한다.
- 채널 1: 유튜브 ACE 공식 채널 — 대형 도달, 일방향
- 채널 2: 네이버 블로그 "연금저축 ETF 추천" — 검색 유입, 신중 선별
- 채널 3: 토스증권 "오늘의 추천" — 즉시 매수 전환, 모바일
- 채널 4: 증권사 PB 대면 상담 — 자산가 자문, 공식성
각 채널별로 최적 이름이 다를 수 있으며, 채널별 가중치를 이름 선정에 반영한다.

### 시나리오 E (이벤트) — 시장 환경별 스트레스 테스트

특정 시장 환경에서 이름이 어떻게 해석되는지 테스트한다.
- E1: 반도체 급락 국면 — "전략"·"주도" 이름이 과장으로 읽히는가
- E2: AI 테마 랠리 국면 — "반차조방원" 이름이 "AI 빠진 올드 ETF"를 더 부각시키는가
- E3: 방산·조선 수주 급증 국면 — "반도체자동차플러스" 이름이 방산·원자력을 누락한 인상을 주는가
- E4: 경쟁사 유사 상품 출시 — 선점 효과가 있는 이름과 없는 이름 구분
- E5: 광고심의 리스크 — "정예"·"주도"·"전략" 등 과장 표현 관련 규제 변수

### 시나리오 H (하이브리드) — 3단 네이밍 조합

정식 상품명 + 서브 캠페인 해시태그 + 숏폼 밈 네임의 3단 조합을 테스트한다. 13개 후보를 슬롯별로 배정해 최적 조합을 도출한다.
- 슬롯 1 (정식명): 방법론 정합성·승인 안전성·공식성 중심
- 슬롯 2 (캠페인): 트렌드 톤·검색 유입·대중 어필 중심
- 슬롯 3 (숏폼 밈): 바이럴·짧음·2030 어필 중심

8) **브랜드 혼동 리스크**: "K5대전략TOP10"의 K5 차량 브랜드 충돌, "K-5대K-TOP10"의 반복 혼란 등 실무 리스크.

11) **이름 단독 매수 전환율**: 각 후보 이름만 보고 매수 의향을 표명한 에이전트 수. 이름의 자립 전환력 측정.

## 최종 리포트 요구사항

본 시뮬레이션의 결과는 단일 HTML 리포트로 산출한다. 리포트 구성은 다음과 같다.

### 섹션 1: 최종 순위표 (Final Ranking)

13개 후보의 1위~13위 순위를 종합 점수와 함께 표로 제시한다. 각 행은 다음을 포함한다.
- 순위
- 후보명
- 종합 점수(가중 평균)
- 유형별 점수 프로필 (10개 유형)
- 채널 가중 점수 (4개 채널)
- 리스크 플래그 (광고심의·브랜드 혼동·중복도)
- 최종 플래그: 🏆 정식명 추천 / 🥈 대체안 / 🎯 캠페인용 / 📱 숏폼용 / ❌ 기각

### 섹션 2: 순위별 이유 설명 (Why This Ranking)

각 순위에 대해 "왜 이 순위인가"를 구체적 근거로 설명한다.
- 1~3위: 각 후보의 강점 3개·약점 2개·타겟 세그먼트·추천 활용 시나리오를 상세 기술
- 4~8위: 간략 설명 + 순위가 낮은 결정적 이유
- 9~13위: 기각 사유 (방법론 위반·심의 리스크·어필 부족 등)
- 각 후보당 대표 에이전트 발화 3개 이상 인용 (긍정·중립·부정)

### 섹션 3: 유형별 1위 후보 매트릭스

10개 투자자 유형 각각의 최선호 후보와 그 이유. 유형 간 합의 후보와 분열 후보를 구분. 합의 후보가 존재하면 전체 1위로 강력 추천, 분열이라면 3단 조합 전략 필요.

### 섹션 4: 극단 후보 분석 (반차조방원 · 반도체자동차플러스)

두 극단 후보에 대해 다른 후보와 구분해 별도 분석.
- 반차조방원: 바이럴 잠재력 vs 공식성 부족·시니어 배제·방법론 위반 → 숏폼 밈 슬롯 전용 추천 여부
- 반도체자동차플러스: 직관성 vs 5섹터 중 2섹터만 노출로 인한 방산·원자력 투자자 소외 → 기각 또는 대조군 위치
- 두 후보가 정식명이 될 경우의 리스크 시나리오

### 섹션 5: 3단 배포 권고안 (Recommended 3-Tier Naming)

정식 상품명 1개 + 서브 캠페인 해시태그 1개 + 숏폼 밈 네임 1개의 최종 조합 추천.
- 슬롯 1 (정식명): 1순위 후보명 + 근거
- 슬롯 2 (해시태그): 트렌드 톤 후보 + 활용 예시
- 슬롯 3 (숏폼): 밈형 후보 + 예시 콘텐츠 컨셉
- 3단 조합이 투자자 유형별 깔때기(인지→관심→탐색→매수)를 어떻게 커버하는지 시각화

### 섹션 6: 이름으로 해결 불가능한 이슈

이름 선정만으로 해결되지 않는 구조적 이슈와 대응 권고.
- AI 부재: 이름에 "AI" 추가는 방법론 위반이므로 상품 구조 변경 또는 서브 상품(ACE K주도주+AI) 출시 검토
- 반도체 50% 집중: 섹터 최소 Cap 10% 도입·반도체 상한 적용 검토
- KODEX 200 중복: 차별화 콘텐츠·백테스트 비교 리포트 상시 운영
- 2차전지·바이오 제외: 방법론 점수 공개로 "숫자상 빠졌다" 설득

### 섹션 7: 시뮬레이션 방법론·한계

- MiroFish 엔진 워크플로우 (그래프 구축 → 페르소나 생성 → 시뮬레이션 → 리포트)
- 사용 모델(GPT-5.2) 및 에이전트 수
- 토큰·비용 요약
- 한계점: 실제 매수 체결 데이터 부재, 이름에 대한 반응이 시뮬레이션 시점 스냅샷임, 경쟁사 대응의 불확실성
- 개선 방향: 실제 카페 투표 데이터와 시뮬 결과 교차 검증, 시드 재주입을 통한 재실행

### 섹션 8: 에이전트 활동 로그 (부록)

13명 투자자 에이전트 각각의 12주 활동 요약, 상위 발화 10건, 타 에이전트와의 상호작용 건수.

## 리포트 형식

- 포맷: HTML, 단일 파일
- 스타일: 기존 `reports/` 폴더의 K수출 리포트와 동일한 다크 테마·표 중심·한국어
- 저장 경로: `reports/ACE_K주도주_이름순위_리포트.html`
- 데이터 시각화: 최종 순위는 막대 차트 또는 점수 히트맵, 유형별 매트릭스는 색상 그리드, 3단 조합은 깔때기 다이어그램

## 실행 순서

1. 시드문서(k주도주(가칭)_시드문서.md)를 MiroFish 프런트엔드에서 업로드
2. 본 요구사항 파일(k주도주(가칭)_시뮬레이션요구사항.md) 내용을 시뮬레이션 요구사항 입력창에 붙여넣기
3. 그래프 구축 → 페르소나 생성 → 시뮬레이션(12주) → 리포트 생성 순으로 실행
4. 최종 HTML 리포트 확인 후 정식 상품명 확정 의사결정 자료로 활용
